Introduction to Large Language Models

A Simple Guide to How LLMs Work, Learn, and Solve Problems

Presenter: Bethanasamy Rajamani
The Paradigm Shift

Traditional NLP vs. LLMs

  • One Model for Many Tasks: Instead of building separate programs for translation, sorting, and summarizing, one LLM handles them all.
  • No Coding Needed to Train: You configure the model by typing plain text instructions (prompts) instead of writing complex code.
  • General Reasoning: The model learns a broad understanding of the world by reading millions of documents.
Real-World Example (Model Efficiency) In the past, a company had to train three different AI models to sort customer emails, extract credit card numbers, and write replies. Today, they replace all three with a single LLM by writing a clear instruction prompt.
Shift in Paradigm

Language models have evolved from simple grammar rules and static translation dictionaries to massive networks. Modern models read text in parallel and weigh how words relate to each other, allowing them to comprehend full contexts rather than individual words.

Comparison: Old models required custom training for every new task; new LLMs run tasks instantly using plain instructions.

Model Architecture

How LLMs Predict Text

  • Smart Auto-Complete: LLMs generate text word-by-word (token-by-token) by guessing the most likely next word.
  • Self-Attention: The model looks at all the words in your prompt to understand how they relate to each other.
  • No Cheating: During learning, the model is blocked from looking ahead, forcing it to practice predicting the next word.
Auto-Complete Example If you type: "The sky is", the model computes the mathematical likelihood of next words and selects "blue". It then takes "The sky is blue" as input to predict the next word.

Data Pipeline

How Computers Read Text

  • Chop Words into Tokens: Computers cannot read text directly. They split words into smaller chunks called "tokens".
  • Root Words: Splitting words helps the computer understand root meanings and deal with spelling errors.
  • Number Mapping (Embeddings): Each token gets a unique ID number, which maps to coordinates representing its meaning.
Tokenization Example The word "unbelievable" is split into subwords ["un", "believ", "able"]. These convert to numbers [102, 1489, 56] so the computer can process them.

Model Training

How LLMs Learn (Pre-training)

  • Reading the Internet: Before an LLM can help you, it goes to school. It reads a massive library of raw web text to learn spelling, grammar, and facts.
  • Self-Practice: It practices by hiding a word in a sentence and trying to guess what word is missing.
  • Creating the Base Model: This creates a "Base Model" that knows facts but doesn't know how to follow instructions yet.
Scale Example A base model reads 3 trillion words. It requires hundreds of powerful servers working together for months. If you prompt it with: "How do I bake a cake?", it might just reply with another question: "How do I bake a pie?" because it hasn't learned to answer yet.
Resource Scale

Pre-training is the most expensive and time-consuming stage. It requires splitting datasets across massive compute networks. The goal is to build general language comprehension, which can later be fine-tuned for specific tasks.

Result: A foundation model containing general grammar rules and global facts.

Optimization Theory

Scaling Laws (Is Bigger Always Better?)

  • Predictable Improvement: You can make a model smarter by increasing its parameter size, feeding it more training text, or using more server power.
  • The Balanced Path: If you make a model 2x larger, you must also feed it 2x more training data to keep it efficient.
  • Emergent Skills: Some skills (like solving riddles or code generation) suddenly appear only when the model crosses a certain size threshold.
Scaling Example Instead of building a giant 100-billion parameter model and training it on a small dataset, developers build a smaller 70-billion model and feed it a much larger dataset. This makes it cheaper to serve while maintaining high accuracy.
Chinchilla Optimal Scaling

In the past, AI builders made models larger without increasing training datasets. Modern scaling laws prove that balancing parameter size and token volume is crucial. A smaller, well-trained model is cheaper to deploy.

Key Takeaway: Optimize for long-term inference cost by training smaller models on more data.

Alignment Stage 1

Teaching the Model to Chat (SFT)

  • Chatbot School: Supervised Fine-Tuning (SFT) teaches the raw base model how to behave like a helpful assistant.
  • Q&A Training Pairs: We show the model thousands of high-quality examples (Prompt: "..." / Answer: "...").
  • Formatting & Tone: Teaches the model to format output as lists, write in polite tones, and obey system rules.
SFT Example A base model completes: "Write a summary of..." with random web sentences. After SFT training on Q&A pairs, it recognizes the request and outputs a structured summary.
Instruction Fine-Tuning Pipeline
Alignment Stage 2

Making Models Safe & Helpful

  • Human Preferences: SFT models might still give bad or unsafe answers. Humans review model replies and vote on which is better.
  • Reward System: We adjust the model weights to reward helpful answers and penalize unsafe ones.
  • DPO Tuning: Direct Preference Optimization simplifies this by updating model weights directly from human choice logs.
Preference Alignment Example A user asks how to bypass security. The model generates two drafts. Humans tag Option A (refusal) as correct, and Option B (bypass instructions) as rejected. The model learns to prioritize helpful and safe boundaries.
Alignment Objectives
Helpful

Answers instructions accurately and assists the user effectively.

Honest

Admits limits, explains when it is unsure, and avoids making up facts.

Harmless

Refuses dangerous requests, filters hate speech, and protects user data.

Prompt Engineering

How to Talk to LLMs (Prompting)

  • Prompt Engineering: The way you ask a question changes the quality of the answer. Clear formatting gets better results.
  • Chain-of-Thought: Asking the model to "explain its reasoning step-by-step" improves math and logic accuracy.
  • System Instructions: Sets overall rules at the beginning of a chat (e.g., "Act as a math teacher").
Step-by-Step Example Prompt: "If John has 5 apples and eats 2, how many left? Explain step-by-step." The model outputs: "1. Start with 5 apples. 2. Subtracted 2 apples. 3. 5 - 2 = 3. Answer: 3." This prevents logical jumps.
Context Constraints

Models are limited by how many words they can process at once (context window). When you paste massive documents, models are very good at reading the start and end of the text, but occasionally miss details in the middle ("lost-in-the-middle").

Tip: Place your most important instructions and contexts at the very beginning or end of your prompt.

Inference Optimization

Making Inference Faster

  • KV Caching (Memory Save): The model remembers keys/values of past words in the chat so it doesn't have to re-read everything at every step.
  • Quantization (Shrink Weights): We compress the model's numbers (from high-precision float to small integers) to fit on cheaper devices.
  • Speculative Decoding: A tiny, fast model drafts text, and the large model approves it.
Optimization Example A standard LLM takes 80ms per word because it recalculates everything from scratch at each step. Enabling **KV Caching** reduces response times to 12ms per word.
Autoregressive Token Generation & KV Loop
Reliability engineering

Why LLMs Lie (Hallucinations)

  • Fluency over Facts: LLMs don't lookup facts in a database; they predict likely words. Sometimes they make up realistic-sounding lies.
  • Grounding (RAG): We lookup actual answers in private files first, paste them into the prompt, and ask the model to summarize them.
  • Zero Temperature: Force the model to choose the most factual words instead of creative ones.
Hallucination Example A chatbot quoting patient data occasionally fabricates medicine names. By pasting the patient's record directly into the prompt and setting temperature=0.0, we force the model to summarize strictly from the text.
Mitigation Pattern

To secure enterprise tools, models should not generate "closed-book" answers for factual query interfaces. The recommended pattern is Retrieval-Augmented Generation (RAG). By supplying source data directly, we turn writing tasks into simple reading tasks.

Target: Minimize hallucinations by replacing open generation with bounded extraction prompts.

Takeaways

Key Takeaways

  • Match Model to Task: Use small models (like Gemma 2B) for cheap, fast local tasks. Save large models (like Gemini Pro) for hard reasoning.
  • Grounding (RAG) First: Use prompt grounding (RAG) when facts change often. Only fine-tune to teach specific formats or tones.
  • Optimize for Speed: Speed up replies using KV Caching and shrink memory usage with 8-bit weight quantization.
  • Safety Boundaries: Deploy API safety filters to detect prompt injections and filter harmful outputs before they reach the user.
Key Lesson On the exam, if a scenario asks to connect private documents with LLM answers without retraining, the answer is always **RAG with Vector Search**.
Core Exam Tip

If the exam asks how to fit a large model on a cheap device with limited memory, choose **Weight Quantization (e.g. to INT8)** to cut memory usage in half.

Practice Scenario 7 of 10

KV Caching & Response Latency

Your online chatbot application gets slower with every new reply. The model has to re-read the entire chat log to predict the next word. Which optimization resolves this latency bottleneck?

A. Scaling up the model parameter size.
B. Enabling Key-Value (KV) Caching to save attention calculations of past words in VRAM memory, bypassing redundant calculations.
C. Lowering the maximum context window size.
D. Using subword tokenization IDs.
Validation Panel
Verify Answer

Correct Answer: B

Explanation: During text generation, models recalculate keys and values for all past words. KV Caching saves these attention tensors in memory during the generation loop, reducing token latency from O(N^2) to O(1) and speeding up responses.

You want to deploy a 30GB model on a cheap edge device with limited VRAM. The device only has 16GB of VRAM available. How can you deploy this model without modifying its layer architecture?
