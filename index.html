Quick Reference

Certification Study Cheat Sheet

Key SQL patterns, Kubeflow DSL commands, Model Monitoring settings, and CLI utilities for quick recall.

BigQuery ML SQL Syntax (Modules 1 & 2)
Standard BQML Syntax
# Create and train a model directly in BQ
CREATE OR REPLACE MODEL `my_dataset.logistic_model`
OPTIONS(
  model_type='logistic_reg',
  input_label_cols=['churned'],
  auto_class_weights=TRUE
) AS
SELECT
  age,
  transaction_count,
  churned
FROM
  `my_project.my_dataset.user_features`;

Know that BQML runs model training natively inside BigQuery's slot architecture. Best for rapid tabular modeling without exporting data.

TF.Transform serving vs Dataflow (Module 2)
  • Dataflow only: Great for preprocessing training data, but leads to skew if serving calculations differ from training.
  • TF.Transform: Outputs a preprocessing graph that is exported *as part of the model*.
  • Key Benefit: Preprocessing logic is identical during batch training and online prediction serving, preventing skew.
Vertex AI Pipelines DSL (Module 4)
KFP DSL python SDK
from kfp.v2 import dsl

@dsl.pipeline(
    name="training-pipeline",
    pipeline_root="gs://my-bucket/root"
)
def pipeline(project: str):
    train_op = custom_train_component(project)
    deploy_op = deploy_model_component(
        model=train_op.outputs["model"]
    )

Kubeflow Pipelines (KFP) compilation outputs a JSON configuration file executed by Vertex AI Pipelines. Serverless, managed execution.

Model Monitoring Settings (Module 5)
  • Training-Serving Skew: Compares serving feature distributions to baseline training data (e.g. TFRecord in GCS).
  • Prediction Drift: Compares active serving inputs day-over-day (detects feature distribution shifts over time).
  • PSI threshold: A standard trigger is PSI > 0.20, indicating substantial distribution shift requiring alerting/retraining.
